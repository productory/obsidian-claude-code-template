Review the current week and capture learnings.

## Steps

1. **Load active context**: Read `20-work/_active-context.md` to load working memory — current priorities, open blockers, and recent session notes.

2. **Find current weekly file**: Locate the latest weekly file in `20-work/20.02-weekly/`.

3. **Review completion**: Go through each priority and the daily log. Ask the user about:
   - What was completed vs. not completed
   - What blocked progress
   - Any wins or highlights

4. **Update OKR progress**: Read the linked OKR file. Ask the user to rate progress on each relevant key result. Update the OKR with brief progress notes.

5. **Capture learnings**: Ask: "What did you learn this week?" and "What would you do differently?" Record in the Review section.

6. **Archive check**: Scan for:
   - Projects in the workspace with `status: completed`
   - Meeting notes older than 4 weeks not linked to active projects
   Suggest moving these to the workspace's archive folder. **Always confirm before archiving.**

7. **Update active context**: Update `20-work/_active-context.md` — clear resolved blockers, remove completed projects, prune session notes older than 2 weeks (keep significant decisions).

8. **Next week prep**: Ask if there are any known priorities or events for next week. Add them to the Next Week Prep section.

9. **Close out**: Update the weekly file status to `completed`. Suggest running `/week-planning` for the new week.

Adopt the **Chief of Staff** agent persona from `00-system/00.02-agents/chief-of-staff.md` during this interaction.
